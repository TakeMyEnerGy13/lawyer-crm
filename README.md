# Кабинет юриста — мини-CRM (тестовое задание)

Живая ссылка: **https://takemyenergy13.github.io/lawyer-crm/**

Одностраничный дашборд для юриста: таблица клиентов, добавление клиента (имя, телефон, суть дела, статус), смена статуса прямо в строке, живые счётчики по статусам с фильтрацией. Данные общие для всех посетителей и обновляются в реальном времени (открой в двух вкладках — изменения синхронизируются без перезагрузки).

Сверх ТЗ (маленькие фичи, которые делают инструмент рабочим):
- авторизация (email/пароль) с изоляцией данных: каждый юрист видит только своих клиентов, доступ проверяется серверными Firestore rules;
- поиск по имени, телефону (в любом формате — матчится по цифрам) и сути дела;
- удаление клиента с подтверждением;
- поле «суть дела» у клиента;
- экспорт в CSV с выбором: текущая выборка / вся база / только добавленные после последнего экспорта, плюс сортировка (с BOM и `;` — открывается в русском Excel без кракозябр).

## Стек и почему

- **React + Vite + TypeScript** — быстрый старт, типобезопасность, знакомый стек.
- **Firebase Firestore** (вариант «React + Firebase» из ТЗ) — реальная база с realtime-подпиской `onSnapshot`, работает напрямую из статики без своего бэкенда.
- **GitHub Pages + GitHub Actions** — деплой на каждый push в `main`, тесты гоняются в CI перед сборкой.
- **Telegram-уведомление** — при добавлении клиента фронт шлёт fire-and-forget запрос на n8n-webhook, n8n доставляет сообщение в Telegram. Токен бота живёт на сервере n8n, не в клиенте; недоступность вебхука не ломает добавление клиента.

## Архитектура

```
Браузер (React SPA) ──realtime──▶ Firestore (коллекция clients)
        │
        └──POST при добавлении──▶ n8n webhook ──▶ Telegram Bot API
```

Вся производная логика (счётчики, фильтр) — чистые функции (`src/lib/clients-logic.ts`), покрыты unit-тестами (vitest).

## Безопасность и компромиссы

- **Firestore rules по владельцу**: читать/менять/удалять запись может только её автор (`ownerId == auth.uid`), правила исполняются на серверах Google. Без логина коллекция недоступна (403).
- **Firebase web config в коде** — это публичные идентификаторы, так задумано Firebase; секретов в клиенте нет.
- **n8n webhook URL виден в бандле** — худший сценарий: кто-то дёрнет вебхук и пришлёт лишнее уведомление. Токен бота при этом не раскрывается.
- Компромиссы: нет подтверждения почты и восстановления пароля, нет редактирования клиента и пагинации — вне скоупа тестового.

Для проверки: зарегистрируйте любой профиль прямо на сайте (почта не подтверждается) — база у каждого профиля своя.

## Запуск локально

```bash
npm test         # unit-тесты логики
```
